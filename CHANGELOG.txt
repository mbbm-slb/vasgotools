vasgotools - Changelog
======================

All notable changes to this project will be documented in this file.

The format is based on Keep a Changelog (https://keepachangelog.com/en/1.0.0/)
and this project adheres to Semantic Versioning (http://semver.org/spec/v2.0.0.html).

## [0.4.0] - 2026-01-06
### Added
- LICENSE file is now embedded and automatically created during app/lib generation with current year
- Cross-platform build scripts (cross-build.bat and cross-build.sh) for building executables for Windows, Linux, and macOS (Intel + Apple Silicon)
- Cross-build scripts are now embedded and automatically deployed when creating apps/libs
- Cross-build scripts are generic and derive binary names from the module name
- Enhanced version detection using runtime/debug.ReadBuildInfo() for accurate version information when installed via go install
- Version command now shows module version from Git tags, commit hash, and VCS modification status

## [0.3.0] - 2026-01-02
### Added
- createOpenVSCodeFile and createBuildScript create artifacts for Windows and MacOS/Linux regardless the current operating system.
- git: commit initial state after module creation
- go-workspace: allow workspace Update
- change the commandline parameters (remove prefix "generate-")
- add analyze scripts and make them part of the module creation
- add README.md
### Fixed
- Fixed problems uncovered by the analyze scripts

## [0.2.1] - 2025-05-12
### Fixed
- use command "go mod edit -go=1.24" to change go version in go.mod from three part format (major.minor.build) to two part format (major.minor). 
  The three part format give an error "go.mod:3: invalid go version '1.24.2': must match format 1.23" on some machines (especially MacOs ?)

## [0.2.0] - 2025-05-12
### Added
- on app creation add the infrastructure to show how version information from git can be shown in the commandline (option -v)
- add usage information to created main.go

## [0.1.1] - 2025-05-09
### Fixed
- do not add the workspace git repository as its own submodule

## [0.1.0] - 2025-05-09
### Added
- initial release for generating apps, libs and workspaces with optionally initializing git repositories
